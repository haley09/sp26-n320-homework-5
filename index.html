GSAP Animation Assignment 5 20 Points

WaveForm — You Got Funded. Now Launch.

The scenario, all five tasks, and the rubric are below. The landing page is live above — scroll through it and watch it transform as you complete each animation task.

📋 THE SCENARIO

Remember BeatFlow? They laid you off. Oh, and by the way — they took your idea, shipped it as their own feature the very next week, and didn't even change the name.

You were furious. You wrote a Medium post. You vented on LinkedIn. And then — you got to work.

Three months later, you have a deck, a prototype, and a meeting with a VC. The meeting goes well. Really well. You just got $500,000 in seed funding to build WaveForm — the music platform you always wanted to build.

Your investors want a public launch page live by Friday. The design team has finished the HTML and CSS. Your job: make it move.

Before You Start: Explore the Files

The landing page above is fully built — all HTML structure and CSS styling is done. Your animations.js file is the only thing you write. Before coding, do this:

  • Scroll through the page slowly — notice there are 5 distinct sections: Hero, Stats ticker, Features grid, Playlist, and Pricing. Each section is one animation task
  • Open DevTools and inspect element IDs — every element you need is already tagged: #heroTitle, #heroLine1, #heroLine2, #featuresGrid, .feature-card, .pl-track, .price-card, and so on
  • Open the browser console and type gsap.to('#heroTitle', {opacity: 0, duration: 1}) — the title should fade out. Type gsap.to('#heroTitle', {opacity: 1, duration: 1}) to bring it back. This confirms GSAP is loaded and working
  • Notice that everything starts invisible — elements have opacity: 0 by default in the CSS. Your animations will fade and slide them in. If the page looks blank, that is correct — your animations.js will reveal everything
  • Read through the animations.js scaffold top to bottom before writing anything — pay attention to which task uses gsap.timeline() vs individual gsap.from() calls

Tasks

Task 1

Hero Section — Timeline Sequence

5 pts

The hero section should reveal itself in a carefully timed sequence when the page loads. Use a GSAP Timeline so each element animates one after another.

  • Create a GSAP timeline: const heroTl = gsap.timeline()
  • Add these animations in order using .from() on the timeline:
    • #heroEyebrow — from opacity: 0, y: -20, duration 0.5s
    • #heroLine1 — from opacity: 0, x: -40, duration 0.6s
    • #heroLine2 — from opacity: 0, x: 40, duration 0.6s, overlapping with previous by "-=0.3"
    • #heroSub — from opacity: 0, y: 20, duration 0.5s
    • #heroActions — from opacity: 0, y: 20, duration 0.4s
    • #heroBadges — from opacity: 0, duration 0.4s
  • After the timeline, animate the three floating cards using gsap.from() with stagger:
    • Target: '.float-card'
    • From: opacity: 0, y: 60, rotation: -5
    • Duration: 0.7s, ease: 'back.out(1.5)'
    • Stagger: 0.15 — so each card animates 0.15 seconds after the previous
💡 Hint: In a timeline, heroTl.from(element, {props}) queues animations sequentially. The "-=0.3" position parameter overlaps the current animation with the previous one by 0.3 seconds — making it feel fluid rather than robotic. Stagger is a property inside the gsap.from() options object: stagger: 0.15.
Task 2

Stats Counter — Number Roll Animation

4 pts

The stats ticker should count up from zero to its final value when it first enters the viewport — a classic "number roll" effect using GSAP's gsap.to().

  • First, fade the stats ticker in: gsap.from('#statsTicker', {opacity: 0, y: 30, duration: 0.6})
  • Create a counter object: const counters = { users: 0, tracks: 0, artists: 0 }
  • Use gsap.to(counters, { ... }) to animate the counter values:
    • Animate to: users: 12400, tracks: 85000, artists: 3200
    • Duration: 2 seconds
    • Ease: 'power2.out'
    • Add an onUpdate callback that runs every frame:
      • Update #numUsers to Math.floor(counters.users).toLocaleString()
      • Update #numTracks to Math.floor(counters.tracks).toLocaleString()
      • Update #numArtists to Math.floor(counters.artists).toLocaleString()
  • Set #numFunding text to "$500K" after a 0.5s delay using gsap.delayedCall(0.5, function() {...})
💡 Hint: gsap.to() on a plain JavaScript object (not a DOM element) animates its numeric properties. The onUpdate function fires every animation frame — use it to read the current interpolated value and write it to the DOM. gsap.delayedCall(seconds, callback) is GSAP's version of setTimeout.
Task 3

Features Grid — Stagger Reveal

4 pts

The 6 feature cards should cascade in with a stagger effect — each card appearing slightly after the one before it.

  • Animate the section title and subtitle first using a short timeline:
    • #featuresTitle — from opacity: 0, y: 30, duration 0.5s
    • #featuresSub — from opacity: 0, y: 20, duration 0.4s, with "-=0.2" overlap
  • Then animate all .feature-card elements together with stagger:
    • From: opacity: 0, y: 50, scale: 0.95
    • Duration: 0.5s, ease: 'power2.out'
    • Stagger: { each: 0.1, from: 'start' } — cards reveal left-to-right, top-to-bottom
  • Add a hover animation on each .feature-card — loop through them with document.querySelectorAll('.feature-card').forEach(card => {...}):
    • mouseenter: gsap.to(card, {y: -6, scale: 1.02, duration: 0.2})
    • mouseleave: gsap.to(card, {y: 0, scale: 1, duration: 0.2})
💡 Hint: stagger: { each: 0.1, from: 'start' } is the object form of stagger — it gives you more control than just passing a number. from: 'start' animates from the first element to the last. Try from: 'center' or from: 'end' to see the difference.
Task 4

Playlist — Slide-In From Left

4 pts

The playlist tracks should slide in one by one from the left side — like a track list loading in real time.

  • Animate the section header using a timeline:
    • #playlistTitle — from opacity: 0, x: -30, duration 0.5s
    • #playlistSub — from opacity: 0, x: -20, duration 0.4s, with "-=0.2" overlap
  • Then animate all .pl-track elements with stagger:
    • From: opacity: 0, x: -60
    • Duration: 0.4s, ease: 'power2.out'
    • Stagger: 0.12 — each track slides in 0.12s after the previous
    • Add a small delay so the tracks start appearing after the header: delay: 0.4
  • Add a click interaction on each track row — when a .pl-track is clicked:
    • gsap.to(track, {backgroundColor: 'rgba(29,185,84,0.15)', duration: 0.2}) to highlight it
    • Use gsap.to(otherTracks, {backgroundColor: 'transparent', duration: 0.2}) to clear the other rows
💡 Hint: For the click interaction, use querySelectorAll('.pl-track') to get all tracks, then inside each click listener, use gsap.to(allTracks, {backgroundColor: 'transparent', duration: 0.2}) to reset everything first, then gsap.to(clickedTrack, {backgroundColor: 'rgba(29,185,84,0.15)', duration: 0.2}) to highlight just the one that was clicked.
Task 5

Pricing Cards — Bounce In + Button Pulse

3 pts

The three pricing cards should bounce in with a playful elastic effect, and the featured card's CTA button should pulse to draw the eye.

  • Animate the section header:
    • #pricingTitle and #pricingSub — fade in from y: 30, opacity: 0 with a short stagger of 0.15
  • Animate the pricing cards with stagger:
    • Target all .price-card elements
    • From: opacity: 0, y: 60, scale: 0.9
    • Duration: 0.6s, ease: 'back.out(1.7)' — this gives the elastic bounce effect
    • Stagger: 0.15
  • Add a repeating pulse animation on the featured card's CTA button:
    • Target: '.price-btn-primary'
    • Use gsap.to() with: scale: 1.05, duration: 0.8, repeat: -1, yoyo: true, ease: 'sine.inOut'
    • repeat: -1 means loop forever; yoyo: true makes it go back and forth
💡 Hint: ease: 'back.out(1.7)' creates an overshoot effect — the element goes slightly past its final position before settling back. The number inside controls how much it overshoots. Try values between 1 and 3 to feel the difference. yoyo: true reverses the animation on every other repeat — perfect for breathing/pulse effects.

Tips for Success

  • 📖 Read the scaffold in animations.js top to bottom — each task section has detailed TODO comments. Read all five before writing a single line
  • 🧪 Test each animation in the console first — GSAP commands typed in the DevTools console run instantly on the live page. Prototype there, then copy to your file
  • ⏱️ Timeline position matters — "-=0.3" overlaps with the previous animation; "+=0.2" adds a gap. Play with these to get timing that feels right
  • 🔁 yoyo + repeat: -1 is your loop pattern — any animation you want to run forever and reverse automatically uses this combination
  • 📐 back.out vs power2.out — use power2.out for smooth, natural motion; use back.out(1.7) when you want a playful overshoot/bounce
  • 💡 onUpdate fires every frame — it's the right place to sync DOM text content with an animating number. Keep it lightweight (no DOM queries inside it)
